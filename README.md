🧠 Reflection Report (≈220 words)

While learning Git, CI, and QA tools, one of the biggest challenges was understanding how Git works, especially branching and resolving merge conflicts. At first, it was confusing to manage multiple versions of the same file and fix conflicts when two changes overlapped. However, with practice, it became easier to understand how branches help organize work and prevent issues.

The CI pipeline using GitHub Actions was very helpful in streamlining the development process. It automatically ran tests every time code was pushed, which helped quickly identify errors without manually testing everything. This saved time and ensured that mistakes were caught early. It also gave immediate feedback on whether the code was working correctly.

Version control and QA practices greatly improved both collaboration and code quality. Using Git allowed multiple people to work on the same project without overwriting each other’s work. Pull requests made it easier to review code and suggest improvements before merging changes. Additionally, writing unit tests and using a linter helped maintain consistent and reliable code. Overall, these tools made the development process more organized, efficient, and professional.
